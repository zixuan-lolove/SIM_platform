## 1. 现状分析

### 1.1 已有展示能力

| 组件 | 能力 | 局限 |
|------|------|------|
| **MapView** (2D 俯视图) | 车辆模型、历史轨迹、参考轨迹、障碍物 | 无路权区域、无规划轨迹对比 |
| **StatusPanel** (状态面板) | 4 组实时数值：位姿/运动/控制/跟踪误差 | 无历史趋势、无阈值告警 |
| **FullStackPanel** (全栈面板) | 任务信息、云端状态、上下行参数、路权分段、驾驶状态、模块状态 | 仅文本标签，无图表 |
| **PlotWindow** (数据曲线) | 4 类信号（车辆状态/位置/控制/跟踪误差）独立图窗，实时滚动 | 信号类别不足，无对比叠加 |
| **DataLogger** | deque 时间序列，按信号名存取 | 无持久化、无导出、无标记 |

### 1.2 缺失的关键能力

1. **测试用例执行视图** — 无法看到"当前在测哪个用例，结果如何"
2. **消息流可视化** — 无法看到 SimMessageBus 上各 topic 的数据流动
3. **状态机可视化** — BusinessDecision 任务状态切换不可见
4. **对比分析** — 无法将参考线 vs 平滑后 vs 规划结果叠加对比
5. **测试判定** — 无自动化的 pass/fail 判断和结果汇总
6. **数据导出** — 无法导出测试数据供离线分析

## 2. 总体架构设计

### 2.1 三层数据展示模型

```
┌──────────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────────┤
│                    第二层：用例详情视图                         │
│  用例描述 | 判定条件 | 实时数据面板 | 关键指标仪表盘            │

│  多信号叠加曲线 | 消息时序图 | 状态转移图 | 数据导出            │

```

## 3. 按测试用例分类的展示方案

### 3.1 T-A1 数据隔离（10条用例）

**核心验证目标**：数据分类路由正确性、时间戳精度、周期性处理、延迟检测、故障处理

#### 3.1.1 新增 UI：消息流监视器 `MessageFlowMonitor`

```
┌─────────────────────────────────────────────────┐
│  消息流监视器                          [暂停] [导出] │
├────────┬────────┬──────────┬─────────┬─────────┤
│ 时间    │ Topic   │ 方向      │ 发布者   │ 订阅者   │
├────────┼────────┼──────────┼─────────┼─────────┤
│ 12.34  │ Lclztn │ Kin→Plan │ Kinemat │ Plannng │
│ 12.35  │ Obs    │ Perc→Plan│ Percept │ Plannng │
│ 12.40  │ PlanRs │ Plan→Ctrl│ Plannng │ Control │
│ ...    │ ...    │ ...      │ ...     │ ...     │
└────────┴────────┴──────────┴─────────┴─────────┘
```

**设计要点**：
- 实时表格 + 瀑布图混合视图
- 每行可展开查看消息体摘要（前 200 字符）
- 按 Topic 过滤 + 高亮异常（延迟 > 阈值变红）
- 统计栏：各 Topic 频率、平均延迟、丢帧数

#### 3.1.2 数据采集需求

| 采集项 | 来源 | 用途 |
|--------|------|------|
| `msg_arrival_ts` | SimMessageBus.publish | 延迟检测 (A1-04) |
| `msg_sequence_id` | SimMessageBus 新增 | 丢帧检测 (A1-03) |
| `msg_source_module` | 发布者标识 | 路由验证 (A1-01) |
| `msg_fields_hash` | 消息体 MD5 | 完整性验证 (A1-10) |
| `fault_injection` | 手动触发标记 | 故障处理 (A1-05~06) |

#### 3.1.3 测试判定面板

```

│  ✓ 所有消息路由到正确的订阅者 (100%)   │

└──────────────────────────────────────┘
```

```python
# sim_platform/core/test_data_collector.py

@dataclass
class TestVerdict:
    """单条判定结果"""
    condition_id: str        # 判定条件编号
    description: str         # 判定条件描述
    expected: str            # 期望值
    actual: str              # 实际值
    passed: bool             # 是否通过

@dataclass  
class TestCaseResult:
    """单条用例执行结果"""
    case_id: str             # 用例编号 (T-A2-03)
    name: str                # 用例名称
    category: str            # 分类 (A1~A6)
    verdicts: list[TestVerdict]
    start_time: float
    end_time: float
    data_snapshot: dict      # 关键数据快照

class TestDataCollector:
    """测试数据采集器"""
    
    def __init__(self, data_logger: DataLogger):
        self._logger = data_logger
        self._active_case: Optional[TestCaseResult] = None
        self._results: list[TestCaseResult] = []
        self._verdict_callbacks: dict[str, callable] = {}
    
    def start_case(self, case_id: str):
        """开始执行一条用例"""
        ...
    
    def add_verdict(self, condition_id: str, passed: bool, 
                    expected: str, actual: str):
        """添加一条判定"""
        ...
    
    def finish_case(self) -> TestCaseResult:
        """结束当前用例，返回结果"""
        ...
    
    def register_auto_check(self, condition_id: str, 
                            check_fn: callable):
        """注册自动判定回调"""
        ...
    
    def export_results(self, path: str) -> None:
        """导出测试结果为 JSON/CSV"""
```
